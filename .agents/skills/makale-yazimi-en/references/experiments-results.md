# Experiments and Results Guide

The Experiments section should convince the reader that the empirical design tests the claims fairly.

## Experimental design checklist

- Research questions are explicit.
- Each RQ maps to a table/figure/result.
- Baselines are justified.
- Metrics match the task and class distribution.
- Statistical or robustness checks are used when needed.
- Negative results are reported honestly.

## Results writing

A result paragraph should:

1. State the main finding.
2. Point to the table/figure.
3. Give the key number(s).
4. Interpret only as far as the evidence allows.
5. Mention exceptions or variability.

## Avoid

- Reading every table cell aloud.
- Hiding weak results.
- Calling marginal differences "substantial".
- Using only accuracy for imbalanced classification.
- Comparing models under inconsistent preprocessing or tuning.
