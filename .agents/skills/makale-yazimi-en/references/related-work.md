# Related Work Guide

Related Work should position the paper. It should not be a list of disconnected summaries.

## Organization options

- By task.
- By method family.
- By dataset or benchmark.
- By evaluation protocol.
- By limitation that motivates the present work.

## Paragraph structure

1. Topic sentence naming the theme.
2. Representative studies and what they establish.
3. Limitation or unresolved issue.
4. Connection to the current paper.

## Good practice

- Group papers by conceptual role.
- Compare assumptions, data, evaluation, and scope.
- Make the gap visible without overstating absence.
- Avoid claiming "no prior work" unless verified.

## Software engineering paper examples

- Defect prediction granularity.
- Cross-project vs within-project prediction.
- Defect labeling and SZZ variants.
- Process metrics vs static metrics vs language-specific metrics.
- Evaluation protocols and leakage control.
