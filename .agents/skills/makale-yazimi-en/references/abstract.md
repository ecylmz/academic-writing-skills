# Abstract Writing Guide

A strong abstract should let a reviewer identify the task, gap, contribution, method, result, and implication in one pass.

## Common structures

### Challenge -> Contribution -> Result

1. Task/context.
2. Specific technical or empirical challenge.
3. Proposed contribution.
4. Method or evaluation setting.
5. Main result.
6. Implication or conclusion.

### Challenge -> Insight -> Contribution -> Result

1. Task/context.
2. Limitation of prior work.
3. Key insight.
4. Method or artifact implementing that insight.
5. Result.
6. Scope/limitation if needed.

### Multiple Contributions

Use this when the paper has several contributions:

1. Task/context.
2. Contribution 1 + advantage.
3. Contribution 2 + advantage.
4. Contribution 3 + advantage.
5. Experimental setting and main results.

## Quality checks

- Does the abstract mention the actual problem solved?
- Is the contribution concrete?
- Are results included?
- Are claims stronger than the evidence?
- Are all acronyms readable?
- Does the final sentence say something specific rather than generic impact?

## Avoid

- "This paper presents a comprehensive study..." without specifying what is studied.
- "State-of-the-art" unless the comparison supports it.
- "Significantly improves" without a metric and baseline.
- Introducing citations unless required by venue.
